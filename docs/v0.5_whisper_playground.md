## рҹҺҜ The Four-Tier Backend Strategy

v0.5 ships with four independent backend families. Pick based on your
network and hardware:

| Tier | Device option | When to use | Speed (demo) |
|------|---------------|-------------|--------------|
| рҹҢҗ **Groq Cloud** | `Groq API` | Online + need fastest response | вҡЎвҡЎвҡЎвҡЎвҡЎ **~64Г— realtime** |
| рҹҢҗ **Fireworks Cloud** | `OpenAI API` (Fireworks endpoint) | Online + Groq quota exhausted | вҡЎвҡЎвҡЎвҡЎ ~29Г— realtime |
| рҹҺ® **Local NVIDIA** | `cuda (faster)` | Offline + NVIDIA GPU (2 GB+ VRAM) | вҡЎвҡЎвҡЎ ~9Г— realtime |
| рҹҸ  **Intel iGPU** | `gpu (OpenVINO-GenAI)` | Offline + Intel iGPU only | вҡЎвҡЎвҡЎ ~7Г— realtime |
| рҹ’» **CPU only** | `cpu (faster)` | Offline + any laptop | вҡЎвҡЎ ~2.7Г— realtime |
| рҹҗҢ **Gemini** | (Gemini endpoint) | вқҢ Not recommended вҖ” interprets instead of transcribing | вҡ пёҸ ~1Г— realtime |

> рҹ’Ў **Demo finding:** the **Grok LPU** (Language Processing Unit, custom
> silicon вҖ” not GPU, not CPU) is what makes Groq's cloud so fast. It's
> *purpose-built* for transformer-style inference.

## рҹӨ– Picking a Model

The **Whisper model** dropdown shows every supported variant:

| Tier | Model | Size | Notes |
|------|-------|------|-------|
| рҹӘ¶ | `tiny` / `tiny.en` | 39 M | Fastest, lowest accuracy |
| рҹҗҰ | `base` / `base.en` | 74 M | Great default for CPU |
| рҹҗӨ | `small` / `small.en` | 244 M | Sweet spot for laptops |
| рҹҰ… | `medium` / `medium.en` | 769 M | High accuracy, slower |
| рҹҰү | `large-v3` | 1.5 GB | Best multilingual quality |
| вҡЎ | `distil-large-v3.5` | ~750 MB | 6Г— faster than large, near-equal accuracy |
| рҹҡҖ | `large-v3-turbo` | ~1.5 GB | **Recommended вҖ” used in the demo** |

> вҡ пёҸ Models ending in `.en` are **English-only**. The Playground shows a
> yellow warning if you pair them with a non-English language.

### рҹ’Ў Why "turbo" in the demo?

On a modest 2 GB NVIDIA card:
- Model size on disk: ~1.6 GB
- VRAM used at runtime: ~1.5 GB (the rest is taken by the display driver)
- Load time: ~3 seconds (warm cache)
- Transcription speed: 18 s of PT-BR audio вҶ’ 7 s on screen вҶ’ **~2.5Г— realtime**

## вҡҷпёҸ Compute Type (Quantization)

Available **Compute Type** values depend on the selected device.

### For `faster-whisper` (CPU / CUDA)

| Type | Use case |
|------|----------|
| `int8` | Smallest, fastest, slight quality drop |
| `int8_float16` | вҡ–пёҸ **Recommended for CUDA** вҖ” best GPU balance |
| `int8_float32` | CPU-friendly |
| `float16` | Standard GPU precision |
| `float32` | Highest precision, slowest |
| `bfloat16` | Modern GPUs (Ampere+) |

### For OpenVINO-GenAI (Intel)

| Type | Use case |
|------|----------|
| `FP16` | OpenVINO default вҖ” full quality |
| `INT8` | вҡ–пёҸ **Recommended** вҖ” half the size, ~same quality |
| `INT4` | Smallest, fastest вҖ” minor quality drop on noisy audio (keep VAD on!) |

### For Cloud

| Type | Meaning |
|------|---------|
| `Groq API` | Just the network call |
| `OpenAI API` | Same, used for OpenAI / Fireworks endpoints |

> рҹ’Ў The combo filters automatically вҖ” you'll never see incompatible options.

## рҹҸ  OpenVINO vs OpenVINO-GenAI вҖ” Important!

From the demo video (paraphrased):

> *"Eu nГЈo aconselho usar [plain OpenVINO] por causa dessa demora. Melhor
> usar este OpenGenie [OpenVINO-GenAI] que ele jГЎ Г© um repositГіrio."*

| Path | Behavior | RAM | Recommendation |
|------|----------|----:|----------------|
| вҡ пёҸ **Plain OpenVINO** | Downloads NVIDIA-format model, **transcodes** to Intel IR locally | ~16 GB | вқҢ Avoid вҖ” slow first run |
| вң… **OpenVINO-GenAI** | Downloads **pre-converted** IR models from HuggingFace | ~1.9 GB | вң… **Use this** |

The OpenVINO-GenAI repo on HuggingFace already contains every model
quantized and ready to load вҖ” no transcoding step, no 16 GB RAM
requirement. **It's a free lunch.**

## рҹ§Ә The Test Panel

This is where you **validate** a backend before saving it.

### Top row

- рҹҢҚ **Language selector** вҖ” `Auto Detect` + popular (PT-BR, EN, ESвҖҰ) +
  a separator + the full 99-language list.
- рҹҺҷпёҸ **Record button** with live VU meter вҖ” push to talk; auto-stops after
  configured silence.
- рҹҺҜ **Status badge** вҖ” `Loading` вҶ’ `Ready` вҶ’ `Cached` вҶ’ `Working` вҶ’ `Reset`.
- вҸұпёҸ **Timer** вҖ” recording elapsed seconds (cap: 30 s per test).

### Status line

Walks you through every phase:
- *"рҹ”ҙ RecordingвҖҰ вҸі loading model in backgroundвҖҰ"*
- *"рҹ”ҙ RecordingвҖҰ вҡЎ model ready (1.2s) вҖ” stop when done."*
- *"рҹ“қ TranscribingвҖҰ"*
- *"вң… Done В· language: pt (99%)"*

### Result + Metrics

After a test, the panel shows:

```
вҸұпёҸ: 1.2s     вҶҗ model load time (or "cached")
рҹҺҷпёҸ: 3.0s     вҶҗ raw audio duration
рҹ“қ: 0.4s     вҶҗ transcription time
вҡЎ: 7.5x     вҶҗ realtime factor
рҹҢҚ: pt: 99% | en: 1%   вҶҗ language probabilities (rolling avg)
вңӮпёҸ: 3.0sвҶ’1.8s (-40%)   вҶҗ VAD savings when applicable
```

> рҹ’Ў The language probabilities are **averaged across all tests in the
> current session** so you can see how stable detection is.

## вҡЎ Smart Caching

The test panel caches the loaded model **across tests** as long as the
config hasn't changed:

- Change just the language? вҶ’ Cached, no reload.
- Change the model or device? вҶ’ Triggers a reload.
- Hit the badge (or Reset) вҶ’ drops the cache, frees memory.

This is why the second, third, fourth tests feel **instant**.

## рҹ“қ Advanced вҖ” Instructions / VAD / Noise

The рҹ“қ button next to the model combo opens an **Instructions** dialog:

| Field | Effect |
|-------|--------|
| **Initial prompt** | Sent to Whisper as context (improves jargon recognition: hostnames, IPs, vendor terms) |
| **No-speech text** | What to show when only silence is detected |
| **Noise level** (1вҖ“99%) | VU meter / recorder threshold |
| **Delay for transcribe** (100вҖ“5000 ms) | Silence duration that ends a recording |
| **вңӮпёҸ VAD pre-trim** | Master on/off |
| **VAD threshold** (0.05вҖ“0.95) | Lower = more sensitive |
| **VAD speech padding** (ms) | Padding around speech to avoid clipping syllables |
| **VAD min silence** (ms) | Minimum silence to split segments |

> рҹҺҜ **Recommended starter values**: VAD on, threshold 0.3, padding 500 ms,
> min silence 500 ms, noise level 5вҖ“10%, delay 1000 ms.

## рҹ’ҫ Save vs Close

- **Save** вҶ’ validates the config, persists to disk, restarts the
  transcription service cleanly (without dropping the toolbar mic).
- **Close** вҶ’ asks if you want to save when dirty (Yes / No / Cancel).
- вқҢ вҶ’ behaves like Close.

> рҹ’Ў **Validation runs first.** If your config is invalid (e.g., a CPU
> threads value > number of cores), you'll see a red error dialog before
> any change touches disk.
